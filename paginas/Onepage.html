DOCKER

O que é DOCKER?

DOCKER é um software de código aberto (código-fonte onde todos podem inspecionar, modificar e aprimorar.) usado para conteinerização virtual, onde permite que vários aplicativos funcionem em diferentes ambientes complexos. De maneira mais simples, constatamos que através disso pode-se utilizar por exemplo o WordPress no Windows, Linux e macOS perfeitamente

Como funciona o DOCKER?

Em sua arquitetura há quatro componentes principais. São eles:

  • Cliente docker: essa arquitetura é o principal método de controle do servidor Docker por meio de uma ILC, como Prompt de Comando (Windows) ou Terminal (macOS, Linux).

  • Servidor Docker (ou daemon): Aguarda as solicitações da API REST feitas pela arquitetura acima e gerencia imagens e containers.

  • Imagens do docker: Instrua o servidor Docker com os requisitos sobre como criar um container Docker. As imagens podem ser tanto personalizadas quanto baixadas de sites (criando um Dockerfile)

    • - Nessa arquitetura é bastante utilazada o DockerHub (hub.docker.com), maior repositório de imagens do Docker

  • Registros do Docker:aplicativo ao lado do servidor utilizado para hospedar ou distribuir imagens do Docker. é muito útil para ter controle sobre elas e armazenagem local.

    • Vantagens e Desvantagens de seu uso

    Apesar de ser muito conhecido, assim como tudo tem prós e contras em seu uso, o Docker não é diferente

    Vantagens:

  • Portabilidade: facilita a instalação/criação de um aplicativo complexo tendo a certeza que ele funcionará

  • Automação: É uma forma de minimizar o gasto de tempo com tarefas chatas e repetitivas, automatizando com auxilio do Cron jobs e containers docker

  • Comunidade: tem um canal dedicado (Slack) fórum e colaboradores de desenvolvimento, além do repositório de imagens

    • Desvantagens:

  • Velocidade: é mais rápido que uma maquina virtual, porém ainda é mais lento que no servidos físico

  • Dificuldade de uso: não é destinado a aplicativos que exijam GUI (Interface gráfica de usuário)

  • Segurança: é executado no Host, fazendo com que qualquer Malware (software malicioso) possa chegar até ele
